FUNDAMENTOS DE LÓGICA DE PROGRAMAÇÃO

O que é Lógica?

A lógica é uma área da Filosofia que se dedica a refletir sobre as diferentes formas de raciocinar. Quando usamos o raciocínio para tomar uma decisão, buscamos sempre a saída mais lógica, ou seja, a opção que nos pareça mais adeqauda para resolver um problema ou atingir um objetivo. A lógica define o encadeamento de ações mais coerentes para chegar a um objetivo. Ela define o uso da razaão para entender, calcular, processar, pesquisar, questionar e decidir sobre as coisas do mundo.

O computador é uma invenção lógica que funciona a partir de uma sequencia de instruções ou comandos. Para que seja eficiente, ele precisa ser programado logicamente, isto é, codificado por meio de elementos e atributos de programação.

Na prática, as sequências lógicas trazem:

  • As instruções que devem ser feitas ("o quê");
  • A orddem em que as instruções devem ser realizadas ("quando")

Em informática, instrução é uma ordem ou um comando que indica ao computador uma ação elementar a ser executada. O computador é uma máquina que realiza exclusivamente o que foi programado para fazer. Ou seja, um programa nada mais é do que uma sequência lógica de instruções organizadas para manipular informações inseridas pelos usuários. Tudo isso é feito para atingir determinado fim.

O que é um algoritmo?

Dentro da lógica de programação, um termo muito utilizado é "algoritmo". Mas você sabe o que ele significa?

O algoritmo não é a solução do problema, mas a descrição detalhadas das etapas que devem ser percorrridas para se chegar a uma solução. O algoritmo tem começo, meio fim e um objetivo a ser alcançado.

Segundo a definição do dicionário, algoritmo é o conjunto de regras e procedimentos lógicos perfeitamente definidos, que levam à solução de um problema por meio de um número finitos de etapas.

Vamos agora conferir quais são as etapas necessárias para o desenvovimento de um programa (sotware):

  • Estudo do problema: análise do problema e de seu contexto para identificar a situação, refletir sobre suas características e apresentar as possível soluções
  • Estruturação do algoritmo: escolha da melhor solução possível para resolver o problema, e estruturação da proposta em forma de "projeto de programa" (algoritmo)
  • Desenvolvimento do código (programação): escrita do algoritmo a partir das regras e dos padrões de uma linguagem de programação especifica. É nessa etapa que ocorre a codificação do algoritmo
  • Implantação da solução: implantação do programa em situação de vida real, para verificar se o problema foi resolvido e a realização de eventuais ajustes (manutenção)

Existe três formas principais de se construir um algoritmo. São elas:

  1. Descrição Narrativa;
  2. Diagrama de Blocos ou Fluxograma;
  3. Português Estruturado ou Pseudocódigo;

uma ordem isolada não permite a realização de um processo completo. Para isso é necessário um conjunto de instruções colocadas em uma sequência lógica.

Programas computacionais são algoritmos que foram codificados em uma determinada linguagem de programação. Existem programas específicos, denominados Aplicativos, que buscam resolver um problema específico a partir de dados informados pelo usuário.

Os programas só resolver situações previamente pensadas. Nesse sentido, programar é prever situações e tomar decisões sobre como alcançar os resultados esperados. Os softwares considerados funcionais e estáveis são aqueles sujos programadores se dedicaram muito em buscar uma sequência lógica perfeita

Vamos conferir as principais regras para o desenvolimento de algoritmos:

  • Seja direto: escreva frases curtas e simples, com apenas um verbo;
  • Seja detalhista: descreva com detalhes todos os passos necessários;
  • Seja lógico: encadeie as ações na ordem na qual elas devem acontecer;
  • Seja objetivo: procure atingir o objetivo proposto com o menor número de passos, tomando o cuidado de não pular passos importantes;
  • Seja claro: evite o uso de palavras e expressões com duplo sentido;
  • Seja compreensível: redija instruções de forma que qualquer pessoa possa entendê-las, não somente profissionais da área de informática;
  • Seja crítico: faça a leitura crítica de seu algoritmo para melhorar a escrita e o encadeamento lógico dele;

Ao desenvolver um algoritmo, devemos submete-lo a rotina elaborao a um processo de verificação, chamado de teste de mesa.

Agora vamos explicar melhor cada tipo de algoritmo que mencionamos anteriormente:

Descrição Narrativa:

A técnica da descrição narrativa pode ser considerada a mais fácil de ser utilizada, pois é a mais prómima da linguagem que usamos no dia a dia para nos comunicarmos.

A descrição narrativa é muito usada em documentos como manuais de equipamentos, receitas culinárias, bula de remédio, descrições do tipo "faça você mesmo", etc

Diagrama de Blocos ou Fluxograma:

O diagrama de blocos é uma forma padronizada e eficaz de representarmos os passos lógicos de um processamento. Nos diagramas de bloco cada ação, instrução ou processamento deve ser descrito dentro de um símbolo geométrico de modo resumido. Setas e palavras são usados para indicar o caminho a ser percorrido, ou seja, o fluxo do processamento.

Confira a seguir os principais símbolos utilizados na construção de fluxogramas

Pseudocódigo, Português Estuturado ou Portugol:

O pseudocódigo usa termos da língua portuguesa encadeados e organizados de forma parecida com as diferentes linguagens de programação.

A opção pelo pseudocódigo na etapa de estruturação do algoritmo proporciona um ganho de tempo na etapa de desenvolvimento do código. É uma forma bastante detalhada de escrever sequências lógica de instrução e processamento.

Confira abaixo as regras para a padronização do pseudocódigo:

  • Todo algoritmo deve ser iniciado com Algoritmo: NomeDoAlgoritmo
  • O início e o fim do programa são limitados pelos marcadores Início e Fim
  • A variáveis são declaradas no início do algoritmo, abaixo do marcador Var, da seguinte forma: NomeDaVariavel: TipoDaVariavel
  • O comando "Leia" deve ser usado para receber (capturar) dados do usuário, fase do processamento conhecida como "Entrada de dados"
  • O comando "Escreva exibe dados para o usuário ("Saída de Dados")
  • Caracteres e Textos (Cadeira ou String) são escritos entre aspas (verificar se a linguagem que você usa utiliza aspas simples ou aspas duplas)
  • Comentário de uma linha podem ser inseridos usando //

Conceitos Fundamentais

Programar computadores requer a definição e o uso de espaços em memória para receber informações inseridas pelos usuários. Os espaços em memória recebem informações que podem variar no decorrer de um processamento. E ai entra outro elemento importante na lógica de programação: as variáveis

Variáveis:

Variáveis são a base da programação imperativa.

Em linhas gerais, variáveis são espaços em memória que servem para guardar um tipo de dado ou informação. Esses espaços em memória devem receber nomes específicos e únicos, para que possam ser referenciados exclusivamente e usados sempres que necessário.

Uma variável representa um endereço da memória RAM (também chamada de memória principal) que armazena, temporariamente, valores e informações. Esses endereços recebem um nome, um tipo e um conteúdo. Cada vez que mencionarmos o nome da variável, seu conteúdo é manipulado

Apesar das linguagens de programação atuais serem mais flexíveis quanto a posição de declaração das variáveis, por uma questão de organização e padronização, é indicado que as variáveis sejam declaradas sempre no início dos algoritmos e dos programas.

Ao longo da rotina estabelecida em um algoritmo ou programa, as variáveis devem receber valores de acordo com o tipo de dado que foi declarado inicialmente. Se declararmos uma variável do tipo caractere, não podemos fazer cálculo com ela, por exemplo. O valor ou conteúdo dessa variável não precisa ser declarado logo quando ela é criada, podemos reservar um espaço na memória principal, definir que tipo de conteúdo ele vai receber, e aguardar que o usuário realize a entrada de dados para o preenchimo do conteúdo.

Existem algumas regras que precisam ser seguidas na hora de nomear as variáveis

  • Nomes de variávels que não podem ser usados:
    • Nome iniciados por número: 1num
    • Que contenham espaço: num 1
    • Que contenham caracteres especiais: número, num*, %num
    • Palavras reservadas: verifiquei quais as palavras reservadas da linguagem escolhida
    • Verifique se a linguagem utilizada é case sensitive, ou seja, se diferencia maiúscas e minúsculas, nesse caso: num1 é diferente de NUM1

    Também é recomendado que os nomes utilizados tenham um relação direta com o significado do dado que ela guarda. Isso facilita o entendimento do código e evita a necessidade de criação excessiva de comentários.

    Tipos Primitivos e Valores

    Basicamente existem 3 tipos principais que se subdividem, e essa divisão pode variar de uma linguagem para outra, por isso, aqui trataremos de um modo mais geral. Esse 3 grupos principais são:

    • Números - geralmente se dividem entre inteiros e reais;
    • Letras - geralmente se divide entre caractere único e cadeia de caracteres (string);
    • Lógico - resposta do tipo verdadeiro ou falso.

    Importante ressaltar que nas linguagens de programação, a parte fracionária dos número reais são delimitadar pelo ponto e jamais pela vírgula (por exemplo, devemos escrever 4.5 e não 4,5). Esse marcador decimal é chamado de "ponto flutuante"

    Operadores

    Operadores são elementos que atuam sobre os operandos (variáveis ou constantes), ou seja, são símbolos ou palavras reservadas por meio das quais é possível fazer cálculos, comprar resultados e atribuir valores às variáveis. Eles são a chave para que possamos incrementar, decrementar, comparar e avaliar dados dentro do computador.

    Precedência geral dos operadores: Parenteses → Aritméticos → Relacionais → Lógicos

    Operadores Aritméticos

    Conforme o próprio nome sugere, esse são operadores que nos permitem realizar cálculos aritméticos, são eles
    + operador de soma
    - operador de subtração
    * operador de multiplicação br / operador de divisão
    % operador para descobrir o resto de uma divisão

    Operadores Relacionais:

    O resultado de qualquer expressão realacional é sempre do tipo lógico.

    A ordem de precedência dos operadores é: NÃO, E, OU

    Operados Lógicos:

    Outros elementos fundamentais na programação são os operadores lógicos. Eles também são conhecidos como dados booleanos, por referência a George Booel, matemático que deu nome à algebra booleana, expressão que trata desses tipos de dados.

    Senquencia de Leitura

    Para que o código seja executado, deve ser seguido uma sequencia de leitura e execussão. Se não existe a presença de estruturas de controle, então a leitura será realizada de forma sequencial. Podemos dividir as estruturas de controle em dois grupos pincipais

    1. Estrutura de seleção. Que por sua vez pode ser:
      • Simples
      • Composta
      • Multipla ou Aninhada
    2. Estruturas de repetição. Divididas em:
      • Para Próximo
      • Enquanto... Faça
      • Faça... Enquanto

    Estruturas Sequenciais

    Estrutura sequencial é uma sequência de instruções que acontecem uma após a outra, sem desvio ou interrupções. Neste caso, o código é composto somente de linhas de comando sucessivas (que compõe um único fluxo possível de processamento)

    Por exemplo:

    Inicio

    num1, num2, resultado: real

    Escreva ("Programa feito para calcular a média do aluno")
    Escreva ("Digite a primeiro nota: ")
    Leia (num1)
    Escreva ("Digite a segundo nota: ")
    Leia (num2)
    resultado = num1 * num2 / 2
    Escreva ("A média é igual a " + resultado)

    Fim

    Estrutura de Seleção

    As estruturas de seleção são usadas para que sejam estabelecidos caminhos diferentes de instrução a serem percurridos a partir de tomadas de decisão. Deverá ser utilizado sempre que tiver de estruturar sequências de ações que poderão ser exercutadas ou não, a depender de um resultado frente a uma ou mais condições.

    Estrutura de Seleção Simples

    A base da estrutura de seleção simples é:
    Se (condição) então
    comandos
    Fim se

    Este recurso deve ser empregado em situações em que se faz necessário testar uma única condição variável, que se for verdadeira, irá desencadear a realização de um ou mais comandos.

    Exemplo:
    Inicio

    num1, num2, resultado: real

    Escreva ("Programa feito para calcular a média do aluno")
    Escreva ("Digite a primeiro nota: ")
    Leia (num1)
    Escreva ("Digite a segundo nota: ")
    Leia (num2)
    resultado = num1 * num2 / 2
    Se (resultado >= 7) então
    Escreva ("Parabéns! Você foi muito bem esse semetre!") Fim se
    Escreva ("A média é igual a " + resultado)

    Fim

    Estrutura de Seleção Composta

    A base da estrutura de seleção composta é:
    Se (condição) então
    comandos
    Senão
    comandos
    Fim se

    Testa uma única condição condição ou variável, se a resposta lógica for verdadeira, irá desencadear a realização de um ou mais comandos, e que se for falsa, irá desencadear outro grupo de ações.

    Exemplo:
    Inicio

    num1, num2, resultado: real

    Escreva ("Programa feito para calcular a média do aluno")
    Escreva ("Digite a primeiro nota: ")
    Leia (num1)

    resultado = num1 * num2 / 2

    Escreva ("Parabéns! Você foi muito bem esse semetre! Sua nota foi: " + resultado)
    Senão
    Escreva ("Você precisa estudar um pouco mais! Sua nota foi: " + resultado)
    Fim se
    Escreva ("A média é igual a " + resultado)

    Fim

    Pode ser feito:

    • Utilizando recurso "se" de forma encadeada ou aninhada (um dentro do outro)

    Estruturas de Repetição

    Estruturas de repetição são necessárias, úteis e importantes no desenvolvimento de rotinas de programação. Com frequência, uma ação ou grupo de ações precisa ser realizado várias vezes. Para resolver esses casos, foram criadas as "Estruturas de Repetição", que permitem criar blocos de repetição até que determinada condição se torne verdadeira, ou por um número predeterminado de vezes.

    Também chamados de "Laços de Repetição", estas estruturas permitem programar o loop (laço, ciclo) de blocos de instrução a partir de parâmetros previamente estabelecidos.

    Formas de controlar o número de vezes que o laço irá se repetir:

    • Contafor fixo: Controla o número de vezes que a estrutura será repetida. Este recurso que utiliza um variável contador para contabilizar o número de vezes que uma determinada sequência será repetida. (contador = contador + 1)
    • Flag de Resposta (sinalização do usuário): Recurso que ao final so processamento, pergunta ao usuário se ele deseja ou não executar a rotina de novo, armazenando o resultado da resposta em uma flag de resposta, isto é, uma variável, por exemplo: resp
      resp = "Deseja continuar <s/n>"
    • Flag Predeterminada (sinalização predeterminada); Recurso que possibilita ao usuário encerrar o loop quando quiser, digitando uma palavra predeterminada, ou quando se chegar a uma condição específica
      "Digite o nome do aluno ou "Fim" para sair"

    Instrução "Enquanto... Faça"

    Indica o início de um loop e recebe uma condição como parâmetro, tal condição é chamada de condição de parada. Nessa estrutura, a rotina de repetição só é executada caso a condição seja verdadeira. Se a condição for falsa, a estrutura é ignorada ou finalizada.

    Exemplo "Enquanto... Faça"
    Início

    num: inteiro
    contador = 1: inteiro
    Escreva ("Esse programa contará de 1 até o número que você escolher")
    Escreva ("Digite até que número você quer contar: ")
    Leia (num)
    Enquanto (contador <= num) faça
    Escreva (contador)
    contador + 1
    Fim Enquanto

    Fim

    Estutura "Para próximo"

    Estrutura básica:
    Para <var=vi> até <vf>[passo <n>]
    Comandos
    Próximo

    Exemplo:
    Para a=2 até 10 passo 2
    Escreva (a)
    próximo

    Nesse caso, o resultado será: 2, 4, 6, 8, 10

    Nesse tipo de construção, a rotina sempre entra na estrutura na primeira vez. Como temos a atribuição var = vi (valor inicial) no início da estrutura, tudo que estiver dentro dela se repetirá até encontrar a palavra "próximo". Quando encontra, a rotina soma o valor do passo (no exemplo citado, somar 2) à varia´vel de controle (a) e realiza um teste com o valor final (vf). Enquanto o valor da variável de controle não atender a condição determinada para o valor final (vf), a rotina continuára se repetindo.

    Esse teipo que estrutura de repetição é indicado quando você já sabe previamente quantas repetição serão necessárias.
