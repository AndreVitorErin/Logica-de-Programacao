Evolução das Linguagens de Programação

Atualmente existem 4 tipos de programação:

  1. Imperativa
  2. Funcional
  3. Orientada a Objeto
  4. Lógica

Para entender como chegamos até aqui, é importante entender a história da programação. Vamos começar essa viagens?

Para entendermos a evolução das linguagens de programação, antes de tudo é necessário compreender a evolução das máquina. Para começarmos a nossa história, vamos voltar até 1801. Nesse ano, na Franças, surgiu um princípo de automação utilizado em teares mecânicos. Joseph Jacquar, através de tabulas bem finas de madeiras perfuradas num padrão específico, faziam com que essas máquinas traçassem desenhos, ou seja, ele "programava" os furos para que os tecidos tivessem desenhos.

O tear em si, obviamente não foi considerado um computador, mas os cartões de madeira perfurados foi utilizado posteriormente no desenvolvimento da programação

Em 1842, Charles Babbage, considerado o pai da computação, idealizou máquinas automatizadas para fazer contas. A primeira que ele criou foi a máquina diferencial, que teve 2 versões.

A máquina diferencial 1.0 podia realizar soma, subtração, multiplicação, divisão e a resolução de polinômios (com poucos dígitos). Mas essa não foi a primeira máquina a fazer contas, aliás, uma anterior foi criada por Pascal, que posteriormente foi homenageado, tendo seu nome atribuído a uma linguagem de programação. Posteriormente, Babbage idealizou uma segunda máquina, mais avançada e maior. Porém, a máquina diferencial 2.0 não pode ser montada por ele, porque não havia recursos tecnológicos na época para que isso fosse possível. Essas máquina foram projetadas usando o sistema decimal. O diferencial da criação de Babbage, é que suas máquinas já tinham o princípio básico de entrada, processamento e saída, e por isso ele é considerado um dos pais da computação.

Posteriormente, ele começou a idealizar uma nova máquina, que ele batizou de "máquina analítica". Ela teria as funcionalidade da máquina diferenciar, mas levando isso a outro nível, ela seria capaz de realizar cálculo de logaritmo, raiz quadrada, em resumo, qualquer coisa que você precisasse. Para que isso fosse possível, seria necessário o uso do cartões perfurados (inventados por Joseph Jacquar) para programar a máquina.

A condessa Ada Augusta Lovelace, que foi aluna de Babbage, auxilou no projeto na máquina analítica e foi a idealizadora do seu sistema de codificação. Ela definou conceitos como: estruturas sequenciais, estruturas condicionais, laços de repetição. Como a máquina analítica nunca ficou pronta, sua linguagem de programção nunca pode ser implementada, mas todos seus princípiosão usados até hoje em todas as linguagens de programação imperativa. Graças a sua contribuição, ela é considerada a primeira programadora da história.

Mas antes de entender os tipos de programação, é importante compreendermos alguns fundamentos básicos

O que é uma Linguagem de programação

Muita pessoa pessam erroneamente que o computador é uma máquina inteligente. Mas na verdade, podemos dizer que o computador é um burro muito rápido. Ou seja, o seu compotador não sabe nada! Mas em compensação, ele é capaz de realizar processamento de dados com uma velocidade impressionante! Portanto é papel do programador escrever as instruções, definindo exatamente o que o computador deve fazer.

Porém, o computador não é capaz de compreender a linguagem humana ou natural. Tudo o que o computador consegue entender é o chamado código binário, também conhecido como bits, linguagem de máquina, ou código executável.

Quando um programador está escrevendo um programa, ele não usa a linguagem natural, mas sim algo que chamamos de linguagem de alto nível, ou linguagem de programação. Mas, como já foi dito, o computador só consegue entender a linguagem de máquina. Portanto, alguém tem que fazer a tradução pra ele! E quem ocupa esse papel são os chamados compiladores.

Na verdade, existem linguagem diferente que usam processos diferentes para realizar esse caminho entre a linguagem de alto nível e a linguagem de máquina (por exemplo, os interpretadores, que passa o código fonte direto para linguagem de máquina, o que economiza memória, mas perde performance), mas para não complicar muito, a princípio vamos generalizar chanado tudo de compiladores

O que é programação?

Em termos gerais, podemos dizer que programação é a arte de resolver problemas. Acredite em mim: aprender a sintaxe de uma linguagem de programação e digitar o código, é questão secundária. O restá desafio é saber resolver problemas utilizando a lógica.

Por isso, é importante se perguntar até que ponto devemos ficar satisfeitos com a primeira solução obtida. Ele é de fato a melhor forma de resolver o problema? Ela poderia ser generalizada para questões semelhantes? Então é importante percebermos a diferença entre problema e instância do problema. Na ciência da computação, estamos interessados em resolver problemas, ou melhor, em estudar problemas, e não comente resolver instâncias dele

O que é um programa?

Um programa é a codificação em alguma linguagem formal que garanta que os passos do algoritmo sejam executados da maneira como se espera por quem executa as instruções. O algoritmo pode ser mais genérico, o programa não

O que é Orientação a Objeto?

Programas orientados a objetos são frequentemente mais fáceis de entender, corrigir e modificar
